1단계
1. 두들점프 클론이다.
2. 싱글플레이부터 구현하고 최종적으로 멀티플레이까지 구현할 것이다.
3. 기본 스택은 HTML+JS이다.
4. 현재 두들점프를 그대로 만든다.
5. 다른 점은 조작방법이다. 틸트 센서값이 아니라 좌우 터치를 얼마나 누르냐로 좌우 방향전환을 한다.


2단계
1. 이동 보조 아이템
- 프로펠러 모자: 머리에 쓰고 헬리콥터처럼 날아오릅니다. 속도는 중간 정도지만 꽤 긴 거리를 안정적으로 올라갑니다.
- 제트팩 : 등 뒤에 메고 강력한 화염을 뿜으며 초고속으로 상승합니다. 게임 내에서 가장 성능이 좋은 아이템입니다.

2. 발판 고정형 장치
- 아이템처럼 인벤토리에 들어오는 게 아니라, 발판 위에 설치되어 있어 밟는 즉시 효과가 나타납니다.
- 스프링 : 발판 위에 놓여 있으며, 밟으면 일반 점프보다 약 2~3배 높이 튀어 오릅 폭니다.
- 트램펄린 : 스프링보다 더 넓은 면적을 차지하며, 밟으면 스프링보다 조금 더 높이 점프합니다.

3. 특수 기능을 가진 발판
- 균열된 발판 : 밟는 순간 부서지며 아래로 추락합니다. 절대 밟으면 안 되는 함정입니다.
- 이동 발판: 파란색은 좌우로, 진한 파란색은 위아래로 움직여 착지 타이밍을 뺏습니다.
- 사라지는 발판: 하얀색 발판으로, 한 번 밟으면 사라지거나 일정 시간마다 깜빡거리며 나타났다 사라집니다.
- 폭발 발판: 밟고 나서 아주 짧은 시간 뒤에발합니다.

3단계
- 타이틀 화면에는 버튼 3개 싱글플레이, 멀티플레이
- 멀티 플레이는 나중에 구현하자 
- 싱글플레이 선택하면 캐릭터 선택화면으로 이동
- 코기, 사람 캐릭터는 각각 2개
- 정사각형 버튼 안에 코기 얼굴 사람얼굴 들어있게 하자.
- 싱글플레이 죽으면 게임종료 화면으로 넘어간다
- 게임 종료화면에는 재시작, 처음으로 버튼 두개 있다.

4단계 멀티 플레이
1. 방식: '로비 및 방 생성 시스템'
2. GUI 화면 설계 
① 메인 로비 (Main Lobby)
- 구성 요소: * 나의 닉네임 표시/수정 칸.
[방 만들기] 버튼.
- 실시간 방 목록 (Room List): 현재 생성된 방들이 카드 형태로 나열됨 (방 제목, 인원수 1/2, 상태).
- [새로고침] 버튼.

② 대기방 (Waiting Room)
- 구성 요소: * 현재 접속한 플레이어 2명의 프로필 (준비 상태 표시).
- 방장 전용 [게임 시작] 버튼 (상대방이 'Ready' 상태일 때만 활성화).
- 간단한 채팅창 (인사 정도 나눌 수 있는 용도).
- [나가기] 버튼.

3. 데이터 구조 (Server-side)
서버(Node.js)에서는 현재 생성된 방들을 객체나 배열로 관리

JavaScript
// 서버에서 관리할 방 데이터 예시
const rooms = {
  "room_123": {
    title: "초보만 오세요",
    players: [
      { id: "socket_A", name: "두들고수", ready: true },
      { id: "socket_B", name: "뉴비", ready: false }
    ],
    isFull: true,
    status: "waiting" // waiting 또는 playing
  }
};
4. 핵심 통신 로직 (Socket.io)
[방 만들기] 과정
- 클라이언트: 서버에 createRoom 이벤트 전송 (방 제목 포함).
- 서버: 고유한 roomID를 생성하고 생성자를 해당 방에 조인시킴.
- 서버: 업데이트된 roomList를 로비에 있는 모든 유저에게 전송.
[방 들어가기] 과정
- 클라이언트: 특정 roomID를 서버에 보내며 joinRoom 요청.
- 서버: 해당 방의 정원(2명)이 찼는지 확인 후 입장 승인.
- 서버: 방 안에 있는 두 사람에게 유저 목록 업데이트 전송.

4. GUI 구현 팁 (HTML/CSS + Phaser)
- 게임을 캔버스(Canvas)로 만들더라도, 로비 UI는 HTML/CSS로 만드는 것이 훨씬 효율적입니다.
- HTML 레이어: 방 목록, 버튼, 채팅창 (디자인과 스크롤 구현이 쉬움).
- Canvas 레이어: 실제 게임 플레이 화면.
- 연결: display: none 속성을 이용해 방에서 게임 시작 버튼을 누르는 순간 HTML 로비를 숨기고 Canvas 게임 화면을 띄웁니다.
